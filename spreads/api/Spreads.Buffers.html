RetainableMemory<T>

Base class for retainable pinned memory. Buffers are always pinned during initialization. Initialization could be from a pool of arrays or from native memory. When pooled, RetainableMemory is in disposed state, underlying arrays are unpinned and returned to array pool.

RetainableMemoryPool<T>

This is thread-safe only with correct usage. With Rent a buffer must be returned via Return and used only by a single thread at a time and no refcounting is done. As soon as a buffer is Retain-ed it must only be used with ref-counting methods such as Increment/Decrement and Dispose of RetainedMemory. A buffer is returned to a pool when refcount reaches zero.
